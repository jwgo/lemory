## 왜 Lemory인가

**측정으로 이긴 검색.** 동일 코퍼스·임베딩·생성기·저지 조건에서 (전체: [BENCHMARKS.md](BENCHMARKS.md)):

| 벤치마크 | **Lemory** | 벡터 RAG | mem0 | cognee | supermemory | qmd |
|---|---|---|---|---|---|---|
| 멀티홉 full-support@8 | **1.000** | 0.561 | 0.579 | 0.561 | 0.579 | 0.526 |
| LOCOMO (LLM 저지) | **0.706** | 0.688 | 0.669¹ | — | — | — |
| 질의 강건성 (한국어/오타/패러프레이즈) | **0.95–0.98** | 0.25–0.49 | — | — | 0.00–0.21 | |
| 검색 지연 (p50) | **~3 ms** | ~1 ms | 212 ms | ~5초 | 327 ms | 0.6–59초 |

**"요새/어제/지난주"를 이해합니다.** 노트마다 날짜를 인식하고(frontmatter › 데일리노트
파일명 › mtime), 사실이 바뀌면 **최신 값이 이깁니다**. 6개월 시나리오 볼트에서 hit@1
**1.000** (벡터 0.273 / BM25 0.636).

**한국어 네이티브.** 조사가 붙어도, 한 글자 단어("책")도 매칭. 영어 노트에 한국어로
질문해도 0.975. 답변은 질문한 언어로.

**키 하나 — 또는 0개.** 무료 Gemini 키 하나로 전부 동작. `pip install "lemory[local]"`
이면 키 없이 완전 오프라인 검색(다국어 로컬 임베딩 220MB).

## 시작하기 (2분)

```bash
pipx install "git+https://github.com/jwgo/lemory"
lemory setup      # 볼트 경로 + Gemini 키 → 진단 + 첫 인덱싱까지 자동
lemory ask "요새 내가 하던 그 프로젝트 어디까지 했지?"
lemory serve      # http://127.0.0.1:8377 웹 UI + 실시간 볼트 감시
```

- **옵시디언에서**: `obsidian-plugin/` 파일 3개를 볼트에 복사 → 사이드바에서 질문,
  출처 클릭하면 해당 노트로 이동
- **Claude Code / VS Code에서**: `claude mcp add lemory -- lemory mcp` (6개 툴)
- **파이썬에서**: `lemory.configure(vault=...); lemory.ask("...")`

## 이런 게 됩니다

- "**아 그거 뭐였지?**" — 모호한 최근 질의, 명시적 기간(어제/지난주/N일 전/3월에),
  바뀐 사실의 최신값 우선
- **오타 나도 찾습니다** — 볼트 어휘 기반 로컬 교정(API 0회), 오타 강건성 0.965
- **목록 질문은 끝까지** — "읽은 책들 전부?"는 검색 깊이가 자동으로 넓어짐
- **이름변경·삭제·별칭·한글 파일명** 전부 워처가 실시간 처리, 오프라인 테스트 236개
- `lemory doctor` 한 방 진단 · `lemory recent` 최근 활동
